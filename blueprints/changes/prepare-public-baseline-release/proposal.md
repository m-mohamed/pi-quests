# Proposal: Prepare public baseline release

## Why

Quest is now in a state where the package, benchmark adapters, and verified
baselines can be shown publicly. The remaining work is not publishing to npm
yet, but tightening the release posture so the repo is easy to evaluate.

## What changes

- codify release readiness as an explicit Quest-native checklist
- record the verified benchmark baseline in a stable doc
- link the release, baseline, and Blueprints workflow from the README
- keep the roadmap for the next benchmark-improvement work in-repo

## Expected outcome

Anyone landing on the repository can answer three questions fast:

- what Quest is
- how it is verified today
- what work is planned next
